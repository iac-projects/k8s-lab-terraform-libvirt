You have to generate a definition file and then add the node port in manually before creating the service with the pod.)

For deployment this will initiate a rolling update according the update strategy (RollingUpdate(default) or Recreate). Current example is for deployment. Remember the --record command-line option you used when creating the Deployment as this will set appropriate versions numebers for your deployments.

kubectl rollout resume deployment nginx Resume paused deployment.
